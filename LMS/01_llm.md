# 生成AIとLLMの“超”基礎をざっくり学ぼう  

Difyのハンズオンに入る前に、「そもそも生成AIって何？」「LLMってどう動くの？」をイメージしてみましょう。

## 1. 生成AIとLLM

生成AI（Generative AI）は、**テキスト・画像・音声などを“新しく作り出す”AI**の総称です。  

> <img src="https://i.gyazo.com/58ed293764534f2782b5728a7616eec2.jpg" width="250px" />  

### 1-1. LLMとは？

生成AIの中でも文章を作ることに特化したAI「**LLM**（Large Language Model／大規模言語モデル）」があります。

> <img src="https://i.gyazo.com/9fa38557e2abdcd459d0e9b841d7e83d.png" width="250px" />  

#### ◾️ LLMはどのように学習しているの？

1. **大量の文章（数兆単語！）** を読む  
2. 「次に来そうな単語」を当てるクイズをひたすら解く  
3. 当たり続けると “文脈” がわかるようになる  

その結果、「質問 ⇒ 文脈に沿った自然な回答」を速く作ることができます。

### 1-2. 代表的なテキスト生成AIサービス

どのサービスも**裏側で「LLM」が動いている**イメージです。

| ツール | 提供元 | 特徴 |
| --- | --- | --- |
| ChatGPT | OpenAI | 対話型LLMの先駆けであり、チャット形式での対話に特化しているがマルチモーダル対応も可。思考を深めた回答をする推論モデルや、AIが自分でWebサイトを見にいき根拠のある情報を元に回答するDeep Research機能もある。 |
| Gemini | Google | テキストや画像、音声など複数の形式を統合して同時に処理できるマルチモーダル対応が強み。Google製品との連携ができるため、GoogleカレンダーやGmailなど実務でも多く利用されている。 |
| Microsoft Copilot | Microsoft | Word / Excel / TeamsなどのM365製品との連携ができる。実務でも使われることが多く、コンテンツの作成やデータ分析などのタスクの自動化が可能。 |
| Claude | Anthropic | 大量の長文テキストやデータの処理が得意。人間味のある自然な対話のやりとりができ、倫理的なガイドラインを設けていることから公平で安全な回答を生成することができる。 |

## 2. LLMの“ありがちトラブル”＝ハルシネーション

文章生成が得意なLLMですが、**ときどきこんな壁**にぶつかります。

| 課題 | 具体例 | ありがちな結果 |
|------|--------|----------------|
| **リアルタイム情報に弱い** | 「今朝の株価は？」 | 古いデータで回答 / ハルシネーション |
| **社内システムにアクセス不可** | 「最新の売上レポートをまとめて」 | 社内データベースを参照できず、対応不可 |
| **実行力ゼロ** | 「この Excel を PDF に変換してメールして」 | 手順の説明はできるが、実行はできない |

このように、壁にぶつかる内容に対しては「それっぽい回答」を返してしまうことがあります。  
この現象を「**ハルシネーション（幻覚）**」と言います。

## 3. LLMを信頼して使うために

LLMの弱点（ハルシネーション）は、「使い方」や「仕組み」で補っていきましょう。

### 3-1. 「使い方」で補う

回答の精度は、「どのように質問するか」で大きく左右されます。  
以下の質問ポイントを意識してみましょう。

- **質問は具体的にする**：曖昧な言葉が多いほど、回答も曖昧になる
- **根拠を意識させる**：「参照元を示して」「理由も書いて」など
- **不明な点を正直に答えさせる**：情報が足りない場合に無理に答えさせない

### 3-2. 「仕組み」で補う

- **プロンプトの工夫**：  
LLMに「どう答えてほしいか」を言葉で具体的に伝える方法。  
手軽に試せるが、できることには限界がある。

- **RAG**：  
LLMに**社内資料や最新情報をあらかじめ用意し、そのデータを参照しながら回答させる**方法。  
ハルシネーションを減らすことができる。

- **ファインチューニング**  
LLMに**専門知識やルールを事前に覚え込ませる**方法。  
毎回同じ基準で答えられるが、準備や更新のハードルが高い。

### 3-3. 「組み合わせて」補う

実際の生成AIサービスではLLM単体ではなく、ツールやシステムと連携して作られています。

- **Web検索/API呼び出し**：最新ニュースをスクレイピング（抽出・整形）、株価APIで数値取得 
- **社内DB/RPA/ワークフロー実行**：社内DBから売上を取得 → LLMに要約させる  
- **ファイル変換・メール送信などのアクション**：ファイルを保存 → 自動でTeams通知させる

## 4. ノーコードで「自分が欲しいAI」を実現する

こうした仕組みは、**コードを書かずにつくることができます。**

| ツール | 何ができる？ | 例 |
|--------|-------------|----|
| GPTs (ChatGPT 内) | オリジナル GPT を GUI で作成。プラグイン／ファイルアップロードが簡単 | 社内 FAQ GPT、英文校正 GPT |
| **Dify** | ノーコードで RAG＋自動化フロー。オンプレ対応・OSS | ナレッジ検索チャット＋ワークフロー |
| Copilot Studio など | 企業向けに業務ツールと連携できる生成 AI | Teams で在庫確認 → 発注 |

## 5. まとめ

初めてAIに触れる方も、「AIって意外と身近で、自分でも作れそう！」と感じていただけていたら嬉しいです。  
普段使っているアプリや街中のサービスなど、身近なところでAIがどう役立っているか探してみましょう。
